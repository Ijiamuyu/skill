# 文档编写规则

通用文档编写规范，适用于项目中的所有文档。

## 核心原则

- **简洁扼要**：用最少的文字说清一件事，删掉一切可有可无的修饰
- **有逻辑**：按因果、流程、分类等逻辑线索组织内容，避免跳跃和堆砌
- **分析说明**：不仅陈述"是什么"，更要说明"为什么"和"怎么做"
- **结构清晰**：善用标题、列表、代码块等格式，让读者一眼把握层次
- **数理表达适度**：用文字和简洁公式解释概念，避免过多公式堆砌；复杂过程用原理图辅助说明，而非纯数学推导

## 格式规范

### 标题层级

```
# 一级标题（文档标题）
## 二级标题（主要章节）
### 三级标题（小节）
```

- 不超过三级标题，过深则说明需要拆分文档
- 标题应直接概括本节内容，避免模糊表述

### 段落

- 一段只说一个主题
- 段落长度不超过 5～7 行，过长应拆分
- 段落间用空行分隔

### 列表

- 并列关系用无序列表 `-`
- 顺序或优先级关系用有序列表 `1.`
- 列表项本身简洁，细节缩进展开

```
- 核心概念：解释 A 是什么
  - 关键特性 1：xxx
  - 关键特性 2：xxx
- 工作原理：说明 A 如何运作
```

### 代码块

- 命令、配置、示例代码用代码块包裹
- 指定语言以启用语法高亮

## 文档结构

一篇好的文档应包含以下要素（视篇幅灵活取舍）：

```
# 标题

## 摘要
用一段话概括全文核心，借鉴学术摘要格式

## 背景 / 问题
说明为什么要写这篇文档，解决什么问题

## 核心内容
按逻辑线索展开正文

## 总结 / 结论
回顾关键结论或下一步行动

## 参考
引用的资料、链接、相关文档
```

### 摘要规范

每篇文章应在标题后、正文前增加摘要，借鉴学术论文摘要格式：

- **背景**：本文讨论什么问题、为何重要（1 句）
- **方法/视角**：从什么角度或用什么框架分析（1 句）
- **发现/结论**：得到了什么关键结论（1–2 句）
- **意义**：这些结论有什么实际价值或启示（1 句）

格式示例：

```
## 摘要

大模型推理成本是制约 AI 应用落地的核心瓶颈。本文从计算
与内存两个维度拆解推理流程，量化分析 Prefill 和 Decode 阶
段的成本构成，并对比主流优化技术（量化、连续批处理、投机
解码）的实际收益。结论是：Decode 阶段的内存带宽瓶颈是效
率关键，连续批处理是当前性价比最高的优化手段。
```

- 摘要控制在 100–200 字，独立成段，不加列表
- 短篇文档（< 200 行）可以省略摘要，或合并到背景中
- 大型文档（跨章节、多级标题）必须包含摘要

### 目录规范

文档应在摘要之后、正文之前插入 `[TOC]` 标记，由 markdown 阅读器自动渲染目录。

- 直接在独立行写入 `[TOC]`，不带额外格式
- `[TOC]` 前后用空行隔开
- 目录层级由阅读器自动生成，无需手动维护锚点链接

格式示例：

```
## 摘要
...

[TOC]

## 背景 / 问题
...
```

- 少于 3 个章节的短文档可省略目录
- 跨章节长文档必须包含目录

## 常用句式规范

| 目的 | 推荐句式 |
|------|---------|
| 说明原因 | 因为……所以…… |
| 对比选择 | 相比 A，B 的优势在于…… |
| 列出步骤 | 首先……其次……最后…… |
| 总结观点 | 综上所述…… |

## 作者标注

- 文档基本上由 AI 创建（无人工大幅改写）→ 文首标注 `> 作者：AI`
- 文档改编自已有材料（书籍、文章、报告等）→ 文首标注 `> 原作者：[材料作者]`
- 文档有明确的人工创作者 → 按用户告知的作者标注
- 多人协作时标注主要贡献者

## 数理表达规范

- **优先用文字解释概念**：能用一句话说清的因果关系，不用公式
- **公式仅用于必须量化之处**：公式应附带文字说明，解释每部分的含义
- **复杂过程用原理图**：用 ASCII 流程图展示流程走向，而非罗列数学推导步骤
- **避免参数堆砌**：表格中的参数值只保留有说明意义的典型值，不列满全部超参数
- **示例**：
  ```
  好：KV Cache 大小随上下文线性增长，以 70B 模型为例，1K→2.7GB，100K→270GB
  差：KV Cache = 2 × seq_len × d_k × n_layers × n_heads × bytes （无解释）
  ```

## 修改规则

- 改动文档时，在文件顶部用 `> 更新于 YYYY-MM-DD` 或类似标记记录变更
- 重大重构（拆分/合并文档）后更新目录索引
- 删除内容前确认没有外部引用指向该内容
- 保持全文语气和术语风格一致

## 示例

```
# 定价模型分析

## 背景
AI 推理成本持续下降，需要一套可量化的定价模型指导服务定价。

## 核心内容
1. **成本构成**：算力 + 存储 + 网络 + 运维
2. **定价策略**：
   - 按 token 计费：单价 × 消耗量
   - 包月套餐：固定费用，用量上限
3. **对比分析**：相比按 token 计费，包月套餐更适合高频用户

## 结论
建议采用混合定价模式：小额用户按 token 计费，大客户走包月套餐。
