# Git 提交规则

通用 Git 提交信息规范，适用于所有项目。

## 格式

```
<类型>: <描述>
```

**类型**：`feat` | `fix` | `docs` | `refactor` | `chore`

**示例**

```
feat: 新增AI推理成本分析文章
fix: 修正定价模型中的数据错误
docs: 更新README使用说明
```

## 原则

- 描述要简洁，一眼看懂变更内容
- 一次提交只做一件事
- 若提交包含多个子项，第一行说明整体任务，后续行缩进对齐，用 `-` 列出子项：

  ```
  feat: 完成用户模块开发
  - 新增注册/登录功能
  - 实现密码加密存储
  - 添加邮箱验证逻辑
  fix: 修复文章模块若干问题
  - 修正分页页码越界
  - 修复Markdown渲染异常
  - 清除未使用的引用
  ```

## 推送规则

- 单人开发，提交后直接推送 `git push origin master`
- 确保推送内容完整、无残留调试代码
